## 第二阶段 几何重建与空间感知 
> **当前进展**：成功将稀疏点云转化为三角网格Mesh，并实现了基于代码的物理碰撞检测

### 1. 过程

#### a. 手动进行数据清洗，去掉远离主体的点云（干扰算法的垃圾点）

#### b. 计算点云法向量 寻找该点的“面”朝向

#### c. 泊松重建：连点成面

---

### 2. 核心技术：泊松表面重建
由于本地硬件为 **ThinkBook 16+ 集显版 (Non-CUDA)**，无法运行耗费显存的稠密点云算法。本项目采用几何推演方案：
* **算法实现**：使用 `Screened Poisson Surface Reconstruction`。该算法通过解泊松方程，将带法向量的点云拟合为连续的曲面。
* **参数调优 (Tuning)**：
    * **Reconstruction Depth**: 设定为 `8`。实验证明，Depth 过高（>10）会引入稀疏点云的空隙噪声，Depth 过低（<6）会导致物体轮廓丢失。
    * **Normal Consistency**: 预处理阶段统一了法向量朝向，解决了重建模型“向内塌陷”的问题。

---

### 3. 空间感知：虚拟体“贴地”行走逻辑
这是本项目向 **空间智能** 迈进的关键一步。我编写了 `walk_test.py` 脚本，赋予了程序“感知”模型表面的能力。

* **技术原理**：**射线检测 (Raycasting)**。从虚拟点的天空位置向下发射向量，计算与 `st.obj` 三角面片的交点。
* **软件工程实践**：
    * 使用 `trimesh` 库加载高效的 **BVH (层次包围盒)** 空间索引，极大提升了碰撞查询速度。
    * **自动地形跟随**：程序能自动获取 Mesh 表面的Y轴高度，模拟了扫地机器人或虚拟 Agent 的避障逻辑。
